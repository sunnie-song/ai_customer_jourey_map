# AI 提效实战 Demo：Excel → Python → HTML

> 一条 AI 生成的自动化工作流——用 Excel 驱动可维护的交互式数据可视化。

## 这是什么

一个面向 **运营 / 产品 / 培训** 场景的 AI 提效实战演示。以「闲鱼用户旅程图」为例，展示如何用 AI 把一页 Excel 数据变成可交互的 HTML 看板，并生成可复用的 Python 转换脚本。

**核心理念**：Excel 是唯一的真相来源，Python 是自动化的桥梁，HTML 是永远新鲜的可交付物。数据更新 ≠ 重新画图。

## 文件结构

```
demo/
├── index.html              ← 分享/培训用的演示页面（浏览器打开即可）
├── 闲鱼用户旅程.xlsx        ← 数据源：卖家旅程 + 买家旅程，可随时编辑
├── generate_html.py        ← Python 脚本：读取 Excel → 生成 HTML
├── journey_map.html        ← 看板交付物：运行脚本产出，直接分享
└── create_excel.py         ← 辅助脚本：生成初始 Excel（可删除）
```

## 快速开始

### 看效果

用浏览器打开 `index.html`，滚动浏览完整的分享演示。

或直接打开 `journey_map.html`，体验交互式用户旅程看板——Tab 切换卖家/买家旅程，查看情绪曲线、痛点、机会点。

### 改数据

1. 打开 `闲鱼用户旅程.xlsx`，在对应单元格修改任意内容
2. 运行转换脚本：

```bash
python generate_html.py
```

3. 刷新 `journey_map.html`——所有修改自动生效

也支持指定文件路径：

```bash
python generate_html.py -i 你的数据.xlsx -o 输出.html
```

### 依赖

```bash
pip install openpyxl
```

## 页面内容说明

`index.html` 按叙事顺序展开：

| 板块 | 内容 |
|---|---|
| **Hero** | 主题：Excel → Python → HTML，AI 生成的顾客旅程 |
| **痛点** | 专业画图工具门槛高 + 效率低、重复工作 |
| **核心思路** | 一条指令 / 三个文件 / 一次搭建 / 永久复用，管线图 + AI 对话示例 |
| **维护对比** | 传统 PPT 修改 30-60 分钟 vs AI 管线 5 分钟 |
| **Demo** | iframe 内嵌 `journey_map.html`，可实时操作切换旅程 |
| **CTA** | 引导思考自己的 Excel 如何变成自动化工作流 |

## 闲鱼用户旅程数据

| 旅程 | 阶段数 | 结构 |
|---|---|---|
| 🏪 卖家旅程 | 7 阶段 | 使用前（决定卖出 → 拍摄上传）→ 使用中（沟通 → 交易 → 发货）→ 使用后（收款 → 评价） |
| 🛒 买家旅程 | 7 阶段 | 使用前（产生需求 → 筛选比较）→ 使用中（沟通 → 下单 → 等收货）→ 使用后（验货 → 评价） |

每个阶段包含：线上/线下行为、触点渠道、用户想法、情绪（高/中/低）、痛点、机会点。
